# unkovenv テストパターン一覧（大分類10）

このファイルは [AGENTS.md](../AGENTS.md) の仕様と [unkovenv](../unkovenv) の現行実装を突き合わせ、
「まず必要な」テストパターンを大分類と小分類で整理したもの。

## 1. CLI・引数解析
- [ ] サブコマンド未指定で usage 表示・終了コード 1
- [ ] 未知サブコマンドで usage 表示・終了コード 1
- [ ] `add` の引数不足/過多で usage 表示・終了コード 1
- [ ] `gc` に余分な位置引数を渡した場合に usage 表示・終了コード 1
- [ ] `status` に余分な位置引数を渡した場合に usage 表示・終了コード 1
- [ ] `--help` / `help` で usage 表示（終了コード 0）
- [ ] `--dry-run` / `--verbose` / `--json` のフラグ受理（サブコマンド横断）

## 2. ストア解決・初期化
- [ ] `UNKOENV_STORE` が最優先で store パスに反映される
- [ ] `UNKOENV_STORE` 未設定時に `UNKOENV_STORE_DIR` が使われる
- [ ] 両方未設定時に `$HOME/.cache/unkoenv` が使われる
- [ ] 空文字 store を検知してエラー終了（終了コード 1）
- [ ] 実行時に `venvs` / `blobs` / `lock` が作成される

## 3. `add` 前提検証（入力妥当性）
- [ ] `venv_dir` 不存在で終了コード 2
- [ ] `venv_dir` は存在するが `site-packages` 未検出で終了コード 2
- [ ] `site-packages` が複数ある場合でも全対象を走査できる
- [ ] 相対パス指定の `venv_dir` を正しく受理・解決できる
- [ ] 空白/日本語を含む `venv_dir` パスで動作する

## 4. ロック・同時実行制御
- [ ] `add` 実行時にロック取得し、成功後にロック解放される
- [ ] `gc` 実行時にロック取得し、成功後にロック解放される
- [ ] 既存ロックがある場合は終了コード 3
- [ ] `--dry-run` 時は実ロックファイルを作成しない
- [ ] 異常終了時もロックが解放される（trap 動作）

## 5. venv エイリアスリンク管理
- [ ] `venvs/<basename>` シンボリックリンクが作成される
- [ ] 既存リンクが同一ターゲットなら成功（冪等）
- [ ] 既存リンクが別ターゲットならエラー終了（終了コード 1）
- [ ] `venvs/<alias>` が通常ファイル等で存在する場合はエラー終了（終了コード 1）
- [ ] シンボリックリンク先が絶対パス（`pwd -P`）で保存される

## 6. 走査対象・除外条件
- [ ] 対象: `site-packages` 配下の通常ファイル（`-type f`）のみ処理される
- [ ] 除外: `__pycache__` 配下が処理されない
- [ ] 除外: `*.pyc` が処理されない
- [ ] 除外: シンボリックリンクは処理対象にならない
- [ ] 深いサブディレクトリ配下の通常ファイルも処理される

## 7. ハッシュ計算・blob 生成/再利用
- [ ] 同一内容ファイルで同一 SHA-256（64桁小文字）に正規化される
- [ ] `blobs/<先頭2文字>/<fullhash>` へ配置される
- [ ] blob 未存在時は新規作成され、`new_blobs` が加算される
- [ ] blob 既存時は再利用される（重複 blob が増えない）
- [ ] ハッシュ計算失敗時にファイルをスキップし、`skipped` が加算される
- [ ] ハッシュツール優先順（`shasum`→`sha256sum`→`openssl`）が機能する

## 8. ファイル置換・ハードリンク整合性
- [ ] blob と元ファイルが既に同一 inode の場合は再置換しない
- [ ] 一時リンク + `mv` によるアトミック置換が成功する
- [ ] 置換後 inode が blob と一致する
- [ ] 既存 blob 利用時に `deduped` / `saved_bytes` が加算される
- [ ] 新規 blob 作成ケースで `saved_bytes` 加算条件が仕様どおり
- [ ] `--dry-run` で実ファイル変更なし・統計のみ更新される

## 9. エラー分類・終了コード
- [ ] 一般エラー（I/O, 権限等）で終了コード 1
- [ ] 前提未満足（venv/site-packages 未検出）で終了コード 2
- [ ] ロック取得失敗で終了コード 3
- [ ] EXDEV（cross-device link）検出で終了コード 4
- [ ] stderr の `[ERROR]` メッセージに原因が含まれる

## 10. `gc` / `status` / 可観測性
- [ ] `gc`: 壊れた `venvs/*` シンボリックリンクが削除される
- [ ] `gc`: `blobs` のリンク数1ファイル（孤立 blob）が削除される
- [ ] `gc`: 空ディレクトリが削除される
- [ ] `gc --dry-run`: 削除を行わずログのみ出力される
- [ ] `status`: `venv_count` / `blob_count` / `broken_link_count` が正しい
- [ ] `status`: `estimated_saved_bytes` がリンク数に応じて計算される
- [ ] `status --json`: JSON 形式で必要キーを出力する
- [ ] 既定ログ/`--verbose` の出力粒度が仕様どおり
- [ ] 再実行時に状態が壊れない（`add`/`gc` の冪等性）
